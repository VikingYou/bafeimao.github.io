从零开始搭建GitHub+HuGo+PicGo图床教程

04.05.2021 11:46 图床 教程 Github 新手 图文

[toc]

安装PicGo

打开网址 向下寻找到稳定版本2.2.2，目前是2.2.2，以后更新自行选择更高稳定版本安装

下载.exe结尾的文件，这里是PicGo-Setup-2.2.2.exe

安装后可能会提示更新，但因为是beta版所以我们选择不更新

下载完成后打开exe文件安装，选择全部用户和单独安装都可以，下一步直到安装结束

打开PicGo显示如下界面

创建Github账号

打开网址 教程网址 如图所示操作

起的名字就是网址地址例如我的是bafeimao则网址地址是bafeimao.github.io以此类推

创建仓库

然后在github个人设置页面，创建一个访问令牌

需要输入密码则输入密码

只勾选第一个repo拉到最下面

点击Generate new token按照下图配置创建即可。

保存生成的密钥

配置PicGo

  • 第一个是用户名/仓库名

  • 第二个是分支名，一般默认设置为main

  • 第三个是Token，就是刚刚拿到的密钥

  • 后面两个可写可不写，建议存储路径写一个，把图片放到一个地方。

  • 点击“确定”，并设置为“默认图床”

上传图片查看github仓库是否有记录

配置Typora+PicGo

Typora

下载安装Typora

打开Typora 左上角 文件->偏好设置->图像

需要勾上插入时自动转义图片URL！

建议都不要勾选，根据使用情况发现手动上传好用一点emmm，可能过段时间就好了。

使用

  • 设置完成后，就可以很方便地使用了。  
  • 拖拽，在文件夹内拖拽图片到Typora编辑界面。
  • 插入，快捷键Ctrl+Shift+I，调出插入图片的功能。
  • 复制，使用截图功能后，直接粘贴到Typora编辑界面内。
  • 拖拽和插入都是自动将图片上传至图床。但是复制需要手动点击上传

安装WindowsTerminal

打开Microsoft Store,搜索terminal并安装

启动后在任务栏右键选择固定到任务栏，以后会常用到，然后关闭窗口右键选择使用管理员模式打开

Set-ExecutionPolicy -Scope LocalMachine -ExecutionPolicy RemoteSigned -Force

复制粘贴后回车执行

然后是下一条命令

Install-Module posh-git -Scope CurrentUser -AllowPrerelease -Force

键入后回车

如果报红出错则键入这个命令再键入上一条命令

Install-Module PowerShellGet -Force -SkipPublisherCheck

之后是下面两条命令配置每次打开终端执行导入git

Import-Module posh-git
Add-PoshGitToProfile -AllHosts

使用命令

git --version

显示出版本则安装成功如下图所示

然后我们来设置一下我们的git信息

设置你的昵称和电子邮箱

git config --global user.name "YOUR NAME"
git config --global user.email "YOUR EMAIL ADDRESS"

在命令行里执行以下命令安装ssh

Install-Module Posh-SSH

输入Y继续然后就会安装成功，输入命令测试

ssh

现在开始安装Hugo

打开hugo网址 选择zip压缩包下载

下载解压到一个文件夹改名字，别放到桌面放到一个安全的地方重命名

里面有三个文件

我们要为他配置全局变量不然只能在这个文件夹下执行命令，使用快捷键Windows+s打开搜索界面键入高级系统设置然后打开如下图所示

需要键入的地址可以在这里复制获得

然后每一步都要确定一共是三个确定直到高级系统设置窗口也关闭，我们关闭命令行重新打开一个输入命令查看

hugo version

hugo安装成功

我们进入到自己想进入的盘符以及想好自己想创建的站点名字

hugo new site Bloghugo

当然也可以在文件浏览器查看我们通过命令行建立的站点

接下来我们学习几个简单的命令行进入和退出文件夹的命令

cd ..
cd yourdocument
ls
clear

ls是查看当前目录的文件比如我在我当前目录使用ls是如下显示

显示的是我当前目录下的文件及文件夹

然后我们可以cd进入一个文件夹或者cd ..进入上一级目录再ls查看当前目录，在这里我们cd到一个目录里，当然不用你把整个目录名都用键盘输入，我们有代码补全键Tab，键入cd s后然后按Tab键，他就会自动补全然后键入回车进入目录

然后使用ls查看，当然我们也可以切换盘符，cd e:命令就可以进入E盘，然后ls查看当前目录

我们cd到主题文件夹里也就是theme

使用git的gitclone命令把主题下载下来

git clone https://github.com/varkai/hugo-theme-zozo

ls可以查看已经下载下来了，然后我们进去看一下把模板拷出来复制到站点指定位置，以下步骤可能使用文件管理器更快速一些

模板里的文件和站点里三个文件是对应的，我们复制粘贴覆盖掉然后在站点目录下，把hugo-theme-zozo文件夹重命名为zozo，运行一下命令在本地执行看看效果

hugo server

注意当前位置

按住Ctrl访问链接查看

本地预览成功接下来我们要推送到github上让他能从网络上访问

由于github时不时被墙所以我们要设置一下代理，防止被墙心情不好还是设置一下比啊较好，很容易暴躁。

git config --global http.proxy 'socks5://127.0.0.1:14808'

git config --global https.proxy 'socks5://127.0.0.1:14808'

git config --global --unset http.proxy

git config --global --unset https.proxy

上面两行命令是设置代理，下面两行命令是取消代理，如果代理不能用或者不需要代理了则可以通过下面两个命令取消,我们也可以通过命令查看我们是否设置代理

git config -l

我这里没有设置是因为通过其他方式翻墙了，啊对，之前的图床如果无法上传图片可能也要设置代理，比较简单就不赘述了都在软件设置里可以找到。

接下来我们正式开始部署到github的操作

打开github登陆自己的账号

这次我们新建的仓库名字要和我们的用户名一致不能不一样，格式就是yourname.github.io,然后下面的应该是public不是private，然后最下面的main也要注意一下有可能是master这会导致以后的命令有一些区别

下面圈出的地方ssh地址后面要用到

然后我们配置ssh让本地电脑可以向github进行推送

打开命令行键入命令生成密钥，我这里用的是自己的邮箱

ssh-keygen -t rsa -C "myname@163.com"

然后一路回车即可

上面的图我其他地方copy来的，大差不差

然后使用下面的命令查看公钥

cat ~/.ssh/id_rsa.pub

会获得ssh-rsa开头邮箱结尾的一长串数字复制下来到github上配置

不用输入Title直接粘贴到key里面然后点Add SSH key就可以然后就变成如上图所示

然后我们现在建立了仓库，配置了SSH，接下来就是去生成我们的文件然后部署到github上面，我们需要对目录下的config.toml文件根据自己的需要更改一些配置，评论系统稍后更新，以后可以根据自己的需要随时更改，更改后打开命令行

键入hugo会构建内容生成一个public文件夹可以进去查看我们使用ls然后cd进去

键入以下命令

git init
git .
git commit -m'test'

然后键入下面一行命令,是一行。

git remote add blog git@github.com:bafeimao/bafeimao.github.io.git

git pull  blog main --allow-unrelated-histories

键入yes

上传成功，可以在仓库里看到新增的文件
